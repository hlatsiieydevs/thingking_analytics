# Changelog: Initial Repository Structure & Specification Setup

## 1. What was updated?
- Created the project directory structure under the monorepo folder `app/` including packages (`shared`, `db`) and services (`ingest-server`, `worker-service`, `ui-service`, `frontend`).
- Generated the architectural blueprint reference file `resources/research/big_picture.md`.
- Updated the root `README.md` with comprehensive description, directory layout, setup instructions, and troubleshooting steps.
- Created `app/.env.example` and `app/docker-compose.yml` for orchestrating the multi-service dev environment.
- Added `.gitignore` and `app/.dockerignore` configurations.

## 2. Why was it updated?
- Establish a clean, scalable, monorepo-friendly project layout in preparation for building the central edge analytics database and gateways.
- Consolidate specifications, communication protocols (MQTT, HTTP), and schemas into a reference document (`big_picture.md`) so that future development stays aligned.
- Containerize system components (PostgreSQL database, RabbitMQ broker with MQTT plugin, Express.js microservices, Vue 3 SPA, Tailscale, Ngrok) to match the target environment.

## 3. How was it updated?
- **Root Workspace configuration**:
  - Initialized `app/package.json` with npm workspaces (`"workspaces": ["services/*", "packages/*"]`).
- **Shared Package (`@thingking/shared`)**:
  - Wrote constants (topics, exchanges, queues) and the abstract class `DataLakeStorage` to support future data lake pluggability.
- **Database Package (`@thingking/db`)**:
  - Formulated the Prisma DB models in `app/packages/db/prisma/schema.prisma` targeting `DwellTimeLog`, `TrafficFlowLog`, and `SystemMetric`.
- **Ingest Server (`@thingking/ingest-server`)**:
  - Coded Express.js REST API with file upload middleware (using `multer`) to receive backlog `.db.gz` files and publish a notification event to RabbitMQ.
- **Worker Service (`@thingking/worker-service`)**:
  - Implemented background consumers for RabbitMQ, handling real-time telemetry updates and unzipping/parsing offline SQLite files to sync PostgreSQL.
- **UI Service (`@thingking/ui-service`)**:
  - Setup WebSocket gateway alongside Express REST routes to broadcast live edge telemetry to browser interfaces.
- **Frontend Dashboard (`@thingking/frontend`)**:
  - Created a Vue 3 Vite application integrated with Tailwind CSS, establishing a WebSocket connection and visual metrics display.
- **Dev-Ops Configuration**:
  - Configured `docker-compose.yml` with dependencies, Dockerfile files, port mappings, and volume mounts.
  - Implemented root-level `.gitignore` and `app/.dockerignore` to ignore node_modules, build outputs, local database files, uploads, and raw data lake directories.
